# Security Policy

CurioQuest is a training project, but the workflow should still be disciplined.

## Supported version

Only the latest release is supported.

## Reporting a vulnerability

Open a private report if your repository supports private vulnerability reporting.
Otherwise, create an issue that avoids exposing exploitable details.

## Security expectations

- Do not commit secrets.
- Do not disable CI just to merge faster.
- Review dependency update pull requests before merging.
- Keep GitHub Actions versions current.

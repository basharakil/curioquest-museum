# Contributing to CurioQuest

Use this workflow after Lab 3:

1. Create an issue with a clear acceptance checklist.
2. Create a branch from `main`.
3. Make a small change.
4. Run `npm run lint`, `npm test`, `npm run build`, and `npm run validate:dist`.
5. Open a pull request and wait for checks.
6. Merge only when the work is understandable and validated.

Good work is boring to review. That is the point.
